# IP Risk Register

This is production guidance, not legal advice.

## Core Rule

Expression rules are **phase-gated** — see the Development Phase Policy in [ip-boundary.md](ip-boundary.md). During the prototype phase, DH-near expression is allowed as temporary placeholder scaffolding in our own original build. **Before any public exposure or ship**, do not reproduce protected expression from Dread Hunger or any other game — originalize it and pass IP Review, using only broad genre structure and emotional goals: cooperative survival, betrayal, proximity voice, shipboard pressure, and hostile environment. Two lines never relax in any phase: never commit/distribute DH files, and never circumvent DRM / extract DH assets.

## Allowed Direction

- Icebound ship survival.
- Near-future fictional icebound research and response vessel.
- Crew-based betrayal survival.
- Proximity voice suspicion.
- Ship maintenance: hull, fuel, power, radio, navigation, hypothermia, flooding, ice pressure.
- Saboteurs as agents or mutineers, not occult magic users.

## Must Avoid

These must not appear in **public-facing or shipped** material. Prototype-phase placeholders are allowed but must be originalized before public exposure (see Development Phase Policy in `ip-boundary.md`).

- Names, logos, store copy, UI layout, map layout, character designs, sound effects, music, and unique terminology from other games.
- Strongly associated terms such as `Dread`, `Hunger`, `Thrall`, `Franklin`, and direct sequel-like naming.
- 19th century arctic expedition ship as the main expression.
- Wooden expedition vessel lookalike.
- Cannibalism, human meat, bone food, poisoned stew, occult totems, blood rituals, curses.
- Iceberg explosion as the primary final objective.
- Same role names, same room arrangement, same victory text, same item names, or same UI composition.
- Marketing language that claims spiritual succession to a specific protected game.

## Safer Expression

- Near-future fictional research vessel.
- Diesel-electric / battery-assisted metal ship.
- Fuel contamination, radio interference, course marker forgery, hatch freezing, hull repair failure.
- Rescue signal, ice pressure route, final navigation sequence.
- Logs, duty records, radio logs, chart notes, tool marks, sensor records.
- Cold-weather work gear, oil, frost, warning LEDs, battery racks, winches, sealed hatches.

## IP Boundary For AI Work

Every AI prompt for **public-facing or ship-bound** design, UI, art, code, or copy must include the line below. For internal prototype scaffolding this is optional — but tag DH-near outputs so the pre-ship originalization pass can find them.

> Do not copy Dread Hunger's names, characters, UI, map, roles, item names, occult elements, cannibalism, Thrall concept, or store wording. Design an original near-future icebound research-vessel survival deduction game focused on hull, fuel, power, radio, route, hypothermia, and proximity voice.

## Copyright Reference

The U.S. Copyright Office states that a game's idea, title, and methods for playing are not protected by copyright, while text and graphic/artistic expression may be protected.

Official references:

- https://www.copyright.gov/register/tx-games.html
- https://www.copyright.gov/help/faq/faq-general.html

## Steam Rights Reference

Steam requires distributed content to comply with Steam agreements and rules. Keep proof that the project owns or has sufficient rights to shipped assets.

Steam Direct Fee and distribution rules entry point:

- https://partner.steamgames.com/doc/gettingstarted/appfee

## Risk Controls

- Maintain an asset ledger.
- Keep AI prompts and generated outputs for shipped AI-assisted assets.
- Prefer licensed Marketplace assets with clear commercial rights.
- Avoid using copied UI screenshots as layout references.
- Run a final IP review before public Steam page, trailer, demo, Playtest, and Early Access.
- Do not use the phrase "spiritual successor" or direct references to another title in store copy.

## Pre-Public Checklist

- Title search completed.
- Store text checked for clone language.
- Capsule art checked for similarity.
- UI checked against direct references.
- Item and role names checked.
- Audio source licenses recorded.
- Fonts recorded.
- Third-party SDK licenses recorded.
