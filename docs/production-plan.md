# Production Plan

## 全体ロードマップ

| Phase | 期間 | 目的 | ゲート |
| --- | --- | --- | --- |
| 0 | 2週間 | 勝てる形を固定 | 企画、GDD、IP、Steam、AI制作体制が文書化済み |
| 1 | 30日 | 8人白箱プロトタイプ | 8人で試合が終わり、再戦意欲が観測される |
| 2 | 3か月 | Vertical Slice | 近接VC、Steam Lobby、主要UI、AI敵、ログ、通報 |
| 3 | 6か月 | Steam Playtest | 100-500人規模の外部テスト |
| 4 | 9-15か月 | Early Access | 安定した8人マッチ、モデレーション、コミュニティホスト |

## Phase 0 完了条件

- 1ページ企画書がある。
- 10ページ以内のGDDがある。
- 類似作分析がある。
- ルール、役職、アイテム、妨害が初期実装粒度である。
- IPリスク方針がある。
- AI生成物の記録方法がある。
- Unreal プロジェクトスケルトンがある。
- Steamworks 登録準備の要件が分かっている。

## Phase 1: 30日白箱

### 1週目: 技術土台と8人接続の成立

成果物:

- Unreal C++ プロジェクトをエディタで開ける。
- 1コマンドでローカルサーバーと8クライアントを起動できる。
- プレイヤー移動。
- インタラクト基盤。
- ドア1種類。
- アイテム取得とドロップ。

完了条件:

- 8クライアントが同一セッションへ参加し、移動、同期、切断をログで確認できる。
- 白箱マップ上でプレイヤー位置、視界、基本インタラクト対象が同期される。
- サーバー権威の最低方針が実装上も守られている。
- SessionId、PlayerId、tick、ping、切断理由がログに残る。

### 2週目: 最小ゲームループ

成果物:

- ロビー。
- 5-8人接続。
- 役職割当。
- 修理タスク。
- 主要区画状態。
- PvE脅威1種類の白箱版。

完了条件:

- ロビー、試合開始、役職配布、試合終了、結果表示の流れが通る。
- 役職は本人だけが確認できる。
- 修理タスクがサーバー権限で完了する。
- 乗員と工作員の仮勝利条件が実装されている。
- 10-15分の白箱試合を最後まで完走できる。

### 3週目: Bot・自動試験・ログ解析

成果物:

- 妨害。
- ダウン/救助。
- 拘束または隔離。
- 勝敗判定。
- 8 bot の自動参加と簡易行動。
- ログ集計スクリプトまたは手順。

完了条件:

- 工作員が勝利条件を満たせる。
- 乗員が勝利条件を満たせる。
- 試合終了が全クライアントに同期する。
- 30分以上の無人連続実行でクラッシュしない。
- ログから試合開始/終了率、平均試合時間、死亡数、切断数、エラー数、勝利陣営、主要イベント時系列を集計できる。
- 重大エラー発生時に再現用 seed、build、log file が紐づく。

### 4週目: 人間プレイテストと継続判断

成果物:

- 6-8人の人間プレイテストを最低2回、可能なら8人テストを5回。
- 全試合録画またはログ保存。
- 面白かった場面と退屈だった場面の整理。
- 次の2か月分の仕様決定。

完了条件:

- 「もう1回」が複数回観測される。
- 重大クラッシュなしで連続3試合できる。
- 削る要素と残す要素が決まる。
- プレイヤーが説明なしで最低限の目的を理解できる。
- 面白さ、混乱、退屈、理不尽さの主要原因が特定される。

## Phase 1 から Phase 2 へ進む条件

### Gameplay Gate

- 8人マッチが最低5回完走する。
- 平均試合時間が35分以下。
- プレイヤーが自発的に疑う、協力する、裏切る場面が発生する。
- PvEが単なる邪魔ではなく、判断を迫る圧力になっている。
- 乗員側と工作員側の勝率が極端に偏っていない。
- 1試合中に明確な山場が最低1回ある。
- プレイヤーの過半数が「もう1回遊びたい」と回答する。

### Technical Gate

- 8人プレイでクラッシュなしの試合完走率が80%以上。
- 重大な同期ズレが既知化され、再現ログが取れる。
- 自動8クライアント試験を定期実行できる。
- 主要イベントログから試合展開を再構成できる。
- パフォーマンス目標を定義し、最低ラインを下回る箇所が把握されている。
- Steam Lobby と Dedicated Server は Phase 2 に送る。Phase 1 は Null/LAN/listen で完走性を優先する。
- 近接VC SDK候補を1つに絞り、Phase 2 の最初に実装検証できる状態にする。

### Production Gate

- Phase 2で作る縦切り範囲が決まっている。
- 削る機能、保留する機能、伸ばす機能が明文化されている。
- アセット台帳が運用されている。
- AI生成物の権利リスクが未分類のまま残っていない。
- 次フェーズの工数見積もりと優先順位がある。

### No-Go 条件

- 試合が頻繁に完走しない。
- プレイヤーが勝敗理由を理解できない。
- 人狼、PvE、サバイバルのどれが主軸か不明。
- ログ不足で不具合を再現できない。
- 主要素材のライセンス状態が不明。
