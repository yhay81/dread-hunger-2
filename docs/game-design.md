# Game Design

> 目標の具体化: 本作は Dread Hunger に近い **ゲーム性・操作性（機能面）** を目標とし、
> **表現（名称・アート・音・テーマ・UI・マップ等）は完全オリジナル** とする。機能（ルール/
> システム/操作）は著作権の保護対象外なので踏襲可、表現は踏襲不可。詳細は
> [Mechanics Parity Target](mechanics-parity-target.md) と [Control Scheme](control-scheme.md)。
> **設定・表現の正典は [`docs/narrative/`](narrative/)**（北極 c.1910s 改装捕鯨船＋顕在型の独自超自然 the Pale）。

## 基本仕様

| 項目 | 初期実装 |
| --- | --- |
| プレイヤー数 | 通常マッチは8人固定。練習プレイは1人 |
| 推奨人数 | 8人 |
| 裏切り者 | 工作員2人 |
| マップ | 近未来の氷海研究船 + 氷上探索エリア |
| 試合時間 | 白箱20-35分、製品18-25分 |

| AI/環境脅威 | 1種類 |

## チーム

### 乗員

船の維持と航路突破が目的。修理、物資運搬、船外探索、航路確認が得意だが、単独行動時に情報が途切れる。

### 工作員

船の到達を阻止する。妨害操作、偽装報告、航路偽装、物資隠し、船外置き去りが得意だが、目撃、修理履歴、行動ログから疑われる。

## マッチ進行

1. 名前付きロビーで8人が集まる。
2. ゲーム開始時に乗員と工作員を秘密裏に割り当てる。
3. 初期状態では船の複数システムが不安定。
4. 乗員は復旧タスクと船外探索を進め、航路進行を完了する。
5. 工作員は妨害、分断、偽情報、事故誘導で進行を止める。
6. ダウン、拘束、死亡が発生すると、残存プレイヤーの判断材料が変わる。
7. 最終航行または船の致命状態で勝敗を決定する。

## マッチング

- ロビー作成時にロビー名を設定する。
- ロビー種別は `casual` と `standard` に分ける。
- `casual` はエンジョイ向けで、参加経験の下限を置かない。
- `standard` は通常向けで、50試合以上の完了経験を目安に参加させる。
- 本番マッチは8人ちょうどで開始し、5-7人開始は開発/検証用の非Shipping設定だけで使う。
- 練習プレイは通常ロビーに掲載せず、1人・工作員なしで開始する。目的は移動、修理、妨害痕跡、船内導線を覚えることに限定する。

## コアシステム

### 船体

船体損傷と浸水が進行する。放置すると区画が危険化し、最終航行に必要な最低状態を満たせなくなる。

### 燃料

航行と発電に影響する。燃料ドラムや冷却剤を運搬し、汚染や漏れを見破る必要がある。

### 発電

照明、無線、ドア、ポンプに影響する。停電時は視界、情報取得、VCの聞こえ方に悪影響を出す。

### 無線

救難信号、航路確認、船外通信に関わる。工作員は周波数ずらしや偽信号で混乱を作る。

### 航路

乗員側の主要勝利進行。海図、観測装置、船首砕氷機構、操舵を組み合わせて氷圧帯を突破する。

> 最終航行（carve-free）の段階/タイマー: Survey→Cut/Charge→Drive（罐を熱く保つ）→Clear（氷圧ゲート突破）。
> **~30分ハードキャップ**で強制決着（停滞＝工作員の時間切れ勝ち）。詳細 `docs/narrative/the-ship.md`（最終航行）/ `docs/narrative/the-pale-and-threats.md` §6。

### 低体温

船外探索と停電区画のリスク。防寒具、医療処置、暖房復旧で対処する。

## 勝利条件の実装しきい値

| 開始人数 | 工作員 | 乗員勝利に必要な船内生存者 | 工作員の排除勝利しきい値 |
| --- | --- | --- | --- |
| 8 | 2 | 5 | 乗員生存3人以下 |

練習プレイでは工作員を割り当てず、1人で航路・修理・船内導線を確認できる。

扱い:

- ダウン中は生存扱いだが、最終航行の勝利人数には数えない。
- 拘束中は生存扱いだが、操舵、無線、修理操作はできない。
- 死亡者はどちらの勝利人数にも数えない。
- 工作員が船内に残っていても、乗員勝利人数には数えない。
- 勝利条件が同 tick で同時成立した場合は、船の致命状態または全滅系の工作員勝利を優先する。
- 最終航行演出開始後に工作員条件が成立した場合、航行完了前なら工作員勝利、完了後なら乗員勝利。

## 初期妨害

詳細は [Sabotage](sabotage.md) を参照。

- 燃料汚染

- 航路マーカー偽装

## 死亡と拘束

Phase 1 では複雑な裁判システムを作らない。優先案は「拘束」。一定時間対象を無力化し、他プレイヤーが救出可能にする。早期死亡で観戦だけになる時間を減らす。

実装優先度:

1. ダウン状態
2. 救助
3. 拘束
4. 隔離
5. 完全死亡

## AI/環境脅威

初期は1種類だけ。候補は「吹雪白outイベント」。理由は、アニメーションコストを抑えつつ、船外探索、方位喪失、声の途切れを作れるため。

> 物語接地: この吹雪 ＝ the Pale の現れ。工作員の **Call**（`docs/narrative/the-pale-and-threats.md`）が局所吹雪を誘発/強化し、脅威と裏切りを結ぶ。
> **白熊（AI 敵）は Phase 2 オプション**（アニメ/AI 予算が許せば、同じ「脅威1種」枠を AI 敵側で満たす案）。

最低機能:

- 一定時間ごとに船外視界を悪化させる。
- 船外VCにノイズまたは距離制限を加える。
- 低体温の進行を早める。
- 工作員がタイミングを利用できる余地を残す。

## 白箱で削る基準

- 会話を生まないタスク。
- 見られても見られなくても結果が同じ妨害。
- 単独行動を過剰に罰して全員団子移動になる仕組み。
- 逆に、集団行動だけが最適になり疑心暗鬼が消える仕組み。
- 死亡後に参加者が退屈になる時間。
