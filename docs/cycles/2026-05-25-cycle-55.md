## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Reuse the same schema validator used by quality gates | This avoids separate schema semantics between CI-style checks and local Windows operations | Keep server config schema changes test-first |
| keep | Fail before launching `AbyssLockServer.exe` on invalid config | Early failure produces clearer Windows handoff evidence than a server boot crash | Capture the validator log in Windows validation notes |
| defer | Full JSON Schema dependency | The current internal validator covers the schema subset used in this repo without adding dependencies | Add a dependency only if schema complexity grows |
