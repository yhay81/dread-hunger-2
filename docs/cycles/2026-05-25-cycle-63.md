## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Put Lobby lifecycle in a `UGameInstanceSubsystem` | Existing telemetry and config systems already use subsystem boundaries, and lobby state spans maps/UI | Implement `UAbyssLobbySubsystem` on Windows after Steam dev config passes |
| keep | UI calls subsystem only | Prevents Blueprint from bypassing build/map validation or writing authoritative data into metadata | Build UI rows after the C++ contract is stable |
| keep | Lobby telemetry logs hashes/reasons, not raw Steam IDs | Diagnostics are needed, but raw platform identifiers should stay out of committed logs | Add hashing/redaction helper before public test telemetry upload |
| defer | Steam Server Browser, SDR, auth, Tool App | These depend on dedicated server and lobby evidence | Continue in P2-006/P2-007 after P2-003/P2-004 |
