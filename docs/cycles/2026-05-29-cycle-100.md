# Cycle 100: match_started / match_ended telemetry carries mode + difficulty (GP-07)

## Header

- Cycle: 100
- Date: 2026-05-29
- Phase: GP-07 (evidence/telemetry) — Madman-mode / host-config follow-up (cycles 97-99)
- Owner: interactive (with owner)
- Goal: Make match lifecycle telemetry self-describing for per-mode analytics. Append `mode` +
  `difficulty` (from the server-authoritative `ActiveMatchConfig`) to every `match_started` and
  `match_ended` event so outcomes can be correlated with the configured mode/difficulty without a
  cross-event join.
- Gate: `quality-gate` green; Unreal **Editor + Game** targets build (GameMode change).

## Planned Scope

- Add a GameMode helper `AppendMatchConfigTelemetry(JsonObject)` that injects
  `"mode":"…","difficulty":"…"` before a payload's closing brace.
- Wrap all 4 `match_started` payloads (dev_auto_start, ready/single_player, host_start, solo_menu)
  and all 4 `match_ended` payloads (timer_expired, fatal_ship_state, crew_threshold,
  final_approach_complete). Auxiliary variant events (`dev_auto_start`, `match_timer_expired`) left as-is.
