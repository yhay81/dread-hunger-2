## Evidence

- Preflight helper: `Tools/playtest_preflight.py`
- Dry-run evidence command passes in dry-run mode and validates required sections, header/summary agreement, and redaction.
- Human-mode command rejects the dry-run example for wrong run id, wrong profile, smoke evidence path, insufficient players, role-count mismatch, dry-run marker, unresolved decision, and unresolved placeholders.
- Raw JSONL input is rejected before any summary checks.

## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Default preflight mode is human | The common path should protect committed P1-024 summaries | Use `--mode dry-run` only for committed smoke examples |
| keep | Check Markdown and summary JSON together | Privacy and unresolved placeholders are only visible in Markdown, while objective gates come from telemetry summary JSON | Run preflight after generating the summary |
| keep | Reject smoke/QA event basis in human mode | Smoke automation cannot count as human social validation | Keep smoke examples separate from P1-024 results |
| change | Subagent-only checks beyond regex remain observer-owned | Understanding, suspicion, consent truth, and feedback fidelity cannot be proven mechanically | Keep those as human checklist items in the packet |
| blocker | P1-024 still needs humans | The validator improves evidence hygiene but does not create social validation | Run the first 6-8 player session when players are available |
