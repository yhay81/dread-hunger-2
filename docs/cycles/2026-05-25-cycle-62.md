## Evidence

- `Tools/ops/lobby_metadata_check.py` rejects:
  - schema violations;
  - `currentPlayers > maxPlayers`;
  - `joinState=open` while full;
  - `joinState=full` while below capacity;
  - raw `IP:port` endpoint tokens;
  - expected build/map mismatch.
- Unit tests cover valid example, build mismatch, and raw endpoint rejection.
- Unit tests also cover inconsistent player count/join-state rejection.

## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Lobby metadata is rendezvous-only | Steam Lobby must not become an authority source for roles, health, inventory, sabotage, or reports | Use this schema as the P2-003 implementation input |
| keep | Build/map mismatch is validated before travel | Failed travel is harder to diagnose than a clear local rejection | Wire the same check into the Windows Lobby spike |
| keep | Endpoint token is opaque | Public IP/port assumptions should stay out until the SDR/server discovery decision | Later map the opaque token to the chosen Steam connection model |
