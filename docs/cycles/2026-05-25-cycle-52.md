## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Use a dedicated interactable actor for life actions | It moves rescue and containment toward the same interaction surface players will use in-game while keeping server authority in character/player-state code | Replace dev-spawned actors with map-placed or component-driven interaction prompts |
| keep | Add `life-action` to the heavier smoke suite | It requires 5 players and validates an end-to-end interaction chain, so it belongs with integration gates rather than unit-only checks | Run it on Windows before production-only feature work resumes |
| keep | Count life-action telemetry in both log and playtest summaries | Playtest reports need to distinguish direct state calls from actual interaction usage | Include this field in future P1 human test report templates |
