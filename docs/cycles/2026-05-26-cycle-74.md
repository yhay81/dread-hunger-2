## Planned Scope

- Add `POST /v1/identity/steam/session-ticket` to the Rust backend.
- Keep the endpoint deterministic and local-only; do not require Steam publisher keys or real Steam credentials.
- Return only `verified`, `ownsApp`, `subjectHash`, `proofId`, and `expiresAt` on success.
- Distinguish missing backend config, invalid ticket, and app-ownership failure.
- Update OpenAPI and backend contract docs so P2-006 is tied to the implemented local mock while real Steam Web API integration remains gated.

## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Mock endpoint requires explicit backend mock config | Missing config must be visible before any caller treats identity as trusted | Wire real config only when Steam Web API credentials are ready |
| keep | Mock ticket sentinels are deterministic and documented | Tests can cover all branches without committing real tickets or publisher keys | Replace sentinel verifier with Steam `AuthenticateUserTicket` and ownership calls later |
| keep | Response excludes raw ticket and SteamID fields | Moderation and fleet callers should consume hashes/proofs only | Extend downstream report/fleet contracts to accept proof IDs after real auth is available |
| defer | Real Steam Web API verification | Requires publisher key storage, environment salt/HMAC handling, retry behavior, and upstream failure UX | Implement behind server-side config after the local contract is exercised from Unreal |
