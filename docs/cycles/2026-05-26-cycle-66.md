## Evidence

- `lobby-join-decision` emits safe fields only: build/map ids, expected ids, player count, join state, connection mode, endpoint availability, `travelAllowed`, and `rejectReason`.
- The command does not print endpoint tokens, raw Steam IDs, auth tickets, IP addresses, player names, or gameplay state.
- The Windows wrapper now records separate logs for `lobby_join_decision`, `lobby_join_build_mismatch`, and `lobby_join_map_mismatch`.

## Decisions

| Label | Decision | Reason | Next Action |
| --- | --- | --- | --- |
| keep | Make join decision a local tool contract first | Steam runtime needs Windows/Steam context, but rejection reasons can be tested safely now | Mirror these reason codes in `UAbyssLobbySubsystem` |
| keep | Treat metadata validation and join rejection as separate checks | Invalid metadata and compatible-but-unjoinable state need different operator evidence | Keep both `lobby-metadata-check` and `lobby-join-decision` in the wrapper |
| defer | Runtime create/find/join | Requires `UAbyssLobbySubsystem`, Steam client, and local Steam dev config | Implement runtime path after Phase 2 entry evidence |
