## Notes / Decisions

- **Metadata is rendezvous-only.** Lobby metadata MUST NOT contain role state, inventory, health, sabotage state, player identity, auth tickets, raw IP addresses, voice data, reports, or match authority. The authoritative state remains in `AFrostwakeGameMode` / `AFrostwakeGameState` / `AFrostwakePlayerState`.
- **No raw endpoint tokens logged.** `lobby-join-decision` and `UFrostwakeLobbySubsystem::EvaluateJoinMetadata` do not emit endpoint token values to logs. `lobbyIdHash` must be a short non-reversible diagnostic token (not a raw Steam Lobby ID).
- **Config safety invariant.** `Config/DefaultEngine.ini` keeps `DefaultPlatformService=Null`. `Frostwake.uproject` keeps `OnlineSubsystemSteam: Enabled: false`. Steam dev work uses only `Saved\Config\steam_dev.local.ini` (gitignored). Do NOT commit this file or any AppIDs, SteamIDs, or publisher keys.
- **Server browser deferred.** Public server browser, Steam Game Servers API registration, SDR model, and Tool App packaging are all deferred until P2-003/P2-004 runtime pass and GP-02 unblocks.
- **GP-02 unblock runbook.** To unblock the runtime spike: set `UE_ROOT` to a source-built/server-capable UE installation, run `cargo run -p frostwake-tools -- unreal-gate --skip-generate --platform Win64 --include-server` to confirm the server target builds, then rerun `run_steam_lobby_validation.ps1` with `-Runtime` and `-SteamConfig Saved\Config\steam_dev.local.ini -RequireSteamConfig`. See `docs/windows-dedicated-server-runbook.md` and `docs/steam-lobby-subsystem-design.md` (Windows Acceptance Commands section).
- **Priority position.** Per `docs/orchestration/PLAN.md` priority order, GP-04 contract/config work ranks 9th. The Next Action here is a low-cost verification gate; it should not consume a full loop iteration if higher-priority lanes (GP-07 health, GP-08, GP-06, GP-03, GP-05, GP-09, GP-10) have eligible work.
