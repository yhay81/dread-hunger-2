## Notes / Decisions

### Confirmed parity findings (read-only analysis, 2026-05-29)

**Gap 1 (Next Action target): `/healthz` `startedAt` field missing from OpenAPI schema.**
- Implementation (`lib.rs` line 328): returns `{ ok, service, version, startedAt }`.
- OpenAPI `HealthResponse` schema: declares only `{ ok, service, version }` as required; no
  `startedAt` property defined.
- Test (`tests/server.rs` `healthz_returns_non_authoritative_service_status`): checks `ok` and
  `service` but never asserts `startedAt`. Fix: add field to schema + add assertion to test.

**Gap 2: Missing 404 on join/leave for unknown lobbyId.**
- `join_lobby` and `leave_lobby` handlers return `(StatusCode::NOT_FOUND, ...)` when the lobby
  is not found.
- OpenAPI paths `/v1/matchmaking/lobbies/{lobbyId}/join` and `.../leave` have no `404` response
  entry.

**Gap 3: Missing 409 on join when lobby is full.**
- `join_lobby` returns `(StatusCode::CONFLICT, ...)` when `members.len() >= required_players`.
- OpenAPI path `/v1/matchmaking/lobbies/{lobbyId}/join` has no `409` response entry.

**Gap 4: Missing 404 on banlist for unknown scope.**
- `get_banlist` returns `(StatusCode::NOT_FOUND, ...)` for scopes not in
  `[official, private_test, local]`.
- OpenAPI path `/v1/banlists/{scope}` has no `404` response entry.

### Authority boundary: CLEAN
- No handler touches movement, combat, role assignment, inventory, sabotage, or match results.
- `x-frostwake-authority: non-authoritative` header is set as middleware on every response.
- Matchmaking lobby backend explicitly does NOT trigger match start; it only tracks slot counts
  and readiness metadata. Unreal owns connection, readiness, role assignment, and match start
  per `docs/backend-contract.md`.

### Data safety: CLEAN
- `RISKY_TEXT_PATTERNS` redaction (IP, SteamID64, email, local paths, secret-like assignments)
  enforced on all mutable text fields in playtest reports, moderation reports, steam identity
  requests, and lobby create/join requests.
- No raw voice, transcripts, SteamIDs, or IP addresses accepted in any endpoint.

### Phase scoping: CORRECT
- Phase 1 endpoints (`GET /healthz`, `POST /v1/playtest-reports`) implemented and tested.
- Phase 2+ endpoints (moderation, identity, matchmaking, banlists, news, fleet) implemented
  as local in-memory prototype, clearly non-authoritative, optional for Phase 1 test runs.
- `docs/backend-contract.md` states: "Keep the backend optional for Phase 1."
