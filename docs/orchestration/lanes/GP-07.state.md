## Notes / Decisions

### What quality-gate currently measures (confirmed from source)

`run_quality_gate` (`crates/frostwake-tools/src/main.rs` line 5446) checks:
`required_files`, `json_parse`, `schema_examples`, `retired_source_files`,
`retired_file_references`, `rust_fmt`, `rust_tests`, `git_ignored`,
`source_control_safety`, `shell_files`, `powershell_files`, `stale_terms`,
`unreal_metadata`, `unreal_cpp_heuristics`, `cycle_records`, `asset_ledger_check`,
`secret_scan`, `git_diff_check`, `git_cached_diff_check`, `unreal_installed`,
`cpp_toolchain`.
No performance, telemetry completeness, or smoke-pass checks are included in the gate.

### Telemetry fields confirmed present in JSONL events

Every event carries: `timestamp_utc`, `sequence`, `elapsed_seconds`, `session_id`,
`run_id`, `build_id`, `map_id`, `profile`, `event`, `payload`.
Source: `Source/AbyssLock/AbyssTelemetrySubsystem.cpp` fn `LogEvent` (line 139).

### Performance metrics: ABSENT from telemetry

No JSONL events currently emit: server tick time, client frame time, ping/latency,
memory usage, or bandwidth consumption. These require UE Stat Unit or a custom
periodic telemetry event. Server-side metrics are additionally blocked by the
Launcher UE server-target limitation. Client/Editor profiling can begin as soon as
a smoke run with `Stat Unit` capture is set up.

### Performance budget: ABSENT from docs

`docs/commercial-quality-target.md` line 46 names the target ("60 FPS client
performance on minimum spec, documented p95 frame time, server tick time, memory,
bandwidth, and load limits") but provides no numbers, thresholds, or measurement
methods. The qa-plan.md playtest template lists "Average FPS" and "p95 ping" as
fields to fill in but they are blank. No standalone performance-budget document
exists anywhere under `docs/`.

### Server-target measurement blocker

`AbyssLockServer.exe` does not exist. Launcher UE distribution does not support
server targets. All server-side performance measurements (tick time, bandwidth
per-connection) are blocked until a source-built or otherwise server-capable UE
is available. Evidence: `docs/cycles/2026-05-26-cycle-78.md` and
`docs/phase1-milestone-report.md`.

### Smoke evidence is gitignored ŌĆö summary docs are the durable record

`Saved/SmokeSuites/` is gitignored. Only committed cycle records
(`docs/cycles/`) and summary docs (`docs/gp03-fatal-sabotage-smoke-summary.md`,
`docs/phase1-milestone-report.md`) provide durable evidence. The M1 gate requires
that every phase gate be reproducible from committed evidence; the current practice
of recording the exact command + build + map + profile + local log path in cycle
records is the correct method, but it depends on cycle docs being kept current.
